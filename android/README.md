# LuaNet for Android

Native Android host application for dedicated Luanti servers. The app targets Android 10+
(`minSdk 29`), ARM64 only, and uses Kotlin, Compose, Room, a foreground orchestrator, and
five process-isolated engine services.

## Build

Install JDK 17 and Android SDK Platform/Build Tools 35, create `local.properties`, then run:

```bash
./gradlew testDebugUnitTest assembleDebug
```

The debug APK is written to `app/build/outputs/apk/debug/`. It is a shell until the native
engine libraries generated by `../engine/scripts/build-version.sh` are copied into
`app/src/main/jniLibs/arm64-v8a/`.

Local debug builds use Android's normal per-user debug key by default. That means an app
installed from this same Windows user can be updated with `adb install -r` without wiping
data.

For shared tester builds or CI artifacts, provide one stable debug keystore explicitly:

```bash
keytool -genkeypair -v -keystore /secure/path/luanet-debug.keystore -storepass luanet-debug \
  -keypass luanet-debug -alias luanet-debug -keyalg RSA -keysize 2048 \
  -validity 10000 -dname "CN=LuaNet Debug,O=NovaX"

LUANET_DEBUG_KEYSTORE=/secure/path/luanet-debug.keystore ./gradlew :app:assembleDebug
```

The optional `LUANET_DEBUG_KEYSTORE_PASSWORD`, `LUANET_DEBUG_KEY_ALIAS`, and
`LUANET_DEBUG_KEY_PASSWORD` values default to `luanet-debug`. Signing keys are ignored by
git and must not be committed. Android cannot update an installed package when the signing
certificate changed, so a device that already has `net.novax.luanet.debug` signed by another
key must either be rebuilt with the old key or backed up/uninstalled and installed once with
the new stable key.

## Runtime boundaries

- `OrchestratorService` owns wake/Wi-Fi locks, the persistent notification, policy limits,
  local ports, and five engine process bindings.
- `EngineSlot0Service` through `EngineSlot4Service` isolate Luanti global native state.
- `NativeEngineBridge` is the stable JNI contract for lifecycle, console, logs, players,
  and crashes.
- The native bridge tails Luanti's per-profile `server.log` and pipes stdin to Luanti, so
  the in-app console is the real dedicated-server console rather than a placeholder queue.
- Each profile is stored below the app-private `servers/<profile-id>` directory.
- ZIP import rejects traversal, links, oversized expansion, excessive compression ratios,
  excessive entries, and invalid package layouts.
- The Content dashboard imports world, game, mod, and modpack ZIP files through Android's
  document picker; imports and manual backups are allowed only while the server is stopped.
- The app always writes Luanti mod security settings and never grants trusted-mod access.

## External services

`AuthTokenProvider`, `AdvertisementGate`, and `BillingGateway` are deliberate integration
boundaries. Production Firebase Auth, AdMob UMP/interstitial, and Play Billing adapters must
be supplied in the Play build after their console resources and secrets exist. LAN hosting
does not depend on those services.

Do not commit `google-services.json`, signing keys, Firebase service accounts, or AdMob IDs.
